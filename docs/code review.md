## 🔴 심각한 문제 (Critical)

### 1. God Object — extension.ts 단일 파일 문제

**현상**: 21,722줄, 365개 함수가 하나의 파일에 존재한다.

**영향**:
- 새 기능 추가 시 어디에 코드를 넣어야 할지 판단 어려움
- 버그 수정 시 부작용 범위 예측 불가
- IDE 자체가 느려짐 (파일 열 때마다 1.1MB 파싱)
- 다른 개발자와 협업 사실상 불가

**특히 거대한 함수들**:
| 함수 | 줄 수 | 문제 |
|------|-------|------|
| `renderRev` | 6,007줄 | 하나의 함수가 6천 줄 — 이해 불가 |
| `deactivate` + OfficePanel | 1,518줄 | 비활성화 로직이 오피스 패널과 혼재 |
| `activate` | 1,154줄 | 진입점이 너무 비대, 모든 커맨드 등록이 여기 |
| `showToast` | 912줄 | 단순 토스트 알림이 900줄 |

**개선 방향**:
```
src/
├── extension.ts          # activate/deactivate만 (~200줄)
├── agents.ts             # 에이전트 정의 (현재 유지)
├── paths.ts              # 경로 관리 (현재 유지)
├── system-specs.ts       # 시스템 스펙 (현재 유지)
├── llm/
│   ├── client.ts         # LLM API 호출 (Ollama, LM Studio)
│   └── models.ts         # 모델 감지/관리
├── brain/
│   ├── brain-manager.ts  # 뇌 폴더 관리
│   ├── rag.ts            # 지식 검색 (RAG)
│   └── git-sync.ts       # Git 자동 동기화
├── agents/
│   ├── ceo.ts            # CEO 분류기/플래너
│   ├── dispatcher.ts     # 스페셜리스트 디스패치
│   └── tools.ts          # 도구 시스템
├── ui/
│   ├── sidebar-chat.ts   # 채팅 사이드바 (SidebarChatProvider)
│   ├── office-panel.ts   # 오피스 패널

│   ├── youtube.ts        # YouTube OAuth
│   └── company.ts        # 회사 설정/메트릭스
└── utils/
    ├── git.ts            # Git 유틸리티

    └── config.ts         # 설정 관리
```

### 2. 테스트 완전 부재

**현상**: 프로젝트에 단위 테스트, 통합 테스트가 전혀 없음.

**영향**:
- 코드 수정 후 기존 기능이 깨졌는지 알 수 없음
- 리팩토링 시 회귀 버그 위험 극도로 높음
- 새 기능 추가의 자신감 없음

**개선 방향**:
- VS Code Extension Testing 프레임워크 도입
- 최소한 핵심 로직(CEO 분류기, RAG 검색, Git 동기화)에 대한 단위 테스트
- LLM 호출은 모킹(mocking)으로 테스트

---

### 3. `any` 타입 남용 (246회)

**현상**: TypeScript의 `strict: true` 설정이 있지만, `any` 타입이 246번 사용됨.

**영향**:
- 런타임 타입 에러 가능성 높음
- IDE 자동완성/리팩토링 도구의 효과 감소
- 버그 탐지 어려움

**개선 방향**:
- 핵심 인터페이스 정의 (LLM 응답, 설정, 대화 메시지 등)
- 점진적으로 `any` → 구체 타입으로 교체
- `unknown` + 타입 가드 패턴 활용

## 🟡 개선 권장 (Warning)

### 4. 인라인 HTML/CSS — 웹뷰 코드 분리 부족

**현상**: 
- `sidebar.html` — 342KB 단일 HTML 파일 (인라인 CSS/JS 포함)
- `extension.ts` 내에 HTML 템플릿 문자열이 대량 포함

**영향**:
- UI 수정 시 TypeScript 파일까지 건드려야 함
- HTML/CSS/JS 각각의 전문 도구(린터, 포매터) 활용 불가

**개선 방향**:
- 웹뷰 HTML → 별도 파일로 분리
- CSS → 별도 파일 (이미 일부는 분리됨)
- JS 로직 → 별도 번들

---

### 5. 매직 넘버 1,373개

**현상**: 의미를 알 수 없는 숫자가 코드 전체에 흩어져 있음.

**예시** (추정):
- 타임아웃: `300`, `240`, `60`, `1500`, `180000`
- 포트: `4825`, `11434`, `1234`  
- UI 수치: 좌표, 크기, 애니메이션 딜레이 등

**개선 방향**:
```typescript
// Before
const r = await axios.get(url, { timeout: 1500 });

// After
const MODEL_DETECT_TIMEOUT_MS = 1500;
const r = await axios.get(url, { timeout: MODEL_DETECT_TIMEOUT_MS });
```

---

### 6. child_process 사용 (25회) — 보안 주의

**현상**: 터미널 명령 실행을 위해 `child_process`가 25회 사용됨.

**긍정적**: `eval()`은 0회 → 코드 인젝션 위험은 낮음  
**주의점**: 사용자 입력이 명령어에 주입될 가능성 검토 필요

**개선 방향**:
- 명령 실행 전 입력 검증/이스케이핑 강화
- 허용 명령어 화이트리스트 적용 고려

---

### 7. 하드코딩된 URL 58개

**현상**: GitHub, Ollama, LM Studio 등의 URL이 코드에 직접 작성됨.

**개선 방향**: 상수 파일이나 설정으로 추출
```typescript
// constants.ts
export const URLS = {
  OLLAMA_DEFAULT: "http://127.0.0.1:11434",
  LMSTUDIO_DEFAULT: "http://127.0.0.1:1234",
  // ...
};
```

---

### 8. VSIX 패키지 크기 (11MB)

**현상**: 빌드된 `.vsix` 파일이 11MB. 주 원인은 pixel 아트 에셋.

**구성 분석**:
- `assets/pixel/` — 수천 개의 이미지 (에이전트 캐릭터, 오피스 타일)
- `assets/agents/` — 프로필 이미지 5개 (5.1MB)
- `out/extension.js` — 1.4MB
- `node_modules/axios` — 2.3MB

**개선 방향**:
- 프로필 이미지 압축/리사이즈
- 사용하지 않는 픽셀 아트 에셋 제거
- `.vscodeignore`로 불필요 파일 제외 (일부 이미 적용됨)
